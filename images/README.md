## Image Guidelines:

- **Format**: JPG or PNG (JPG preferred for photos, PNG for graphics with transparency)
- **Quality**: High resolution, but optimized for web (under 300KB each)
- **Aspect Ratio**: 2:1 ratio (600x300px) works best for blog cards
- **Professional**: All images should look professional and relevant to the article topic

## Blog Image Best Practices:

- Use consistent styling across all blog images
- Ensure text overlays are readable if used
- Consider using tools like Canva or Figma for creating custom graphics
- Keep file sizes optimized for fast loading on GitHub Pages
- Use relevant imagery that represents the blog post content

## GitHub Pages Optimization:

Since you're hosting on GitHub Pages:
- Keep total image folder size under 100MB
- Optimize images before uploading
- Consider using WebP format with JPG fallback for better compression
- Test loading speed after adding all images

## Placeholder Images:

If you don't have blog images ready yet, you can use:
- https://picsum.photos/600/300 (random photos)
- https://via.placeholder.com/600x300 (solid color placeholders)

Just make sure to replace them with your actual blog-related content before going live!
